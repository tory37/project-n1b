Reading List

Resources to read before implementing networking.

Godot 4 Networking

1
High-level multiplayer — Godot Engine docs
docs.godotengine.org

Establishes core vocabulary: peers, peer_id, server/host, authority. Read slowly — everything else builds on this.

2
Building an Authoritative Server from Scratch — StraySpark
strayspark.studio

Shows authority, ENet peers, and RPCs wired together. Explains who the authority is and why it matters.

3
RPC in a turn-based multiplayer game — Godot Forum
forum.godotengine.org

Directly relevant to this project's architecture: discrete events, not continuous state.

4
Multiplayer in Godot 4.0: RPC syntax — Godot Engine blog
godotengine.org

Reference for @rpc("any_peer") and @rpc("authority") annotation options. Bookmark and refer back.

Note

For a turn-based card game, MultiplayerSynchronizer and MultiplayerSpawner are not needed — those are for continuously syncing real-time state like player position. This project only needs ENet + RPCs.
