State Mutation Pattern

The networking seam that all game actions follow.

sequenceDiagram participant UI as UI / World participant C as GameManager (Client) participant S as GameManager (Server) Note over UI,S: Any state mutation — spend AP, draw card, pass turn… UI->>C: SignalBus.*_requested (signal up) C->>C: _on_*_requested() C-->>S: request_*.rpc_id(1, …) S->>S: _apply_*() — validate + mutate S-->>C: _rpc_sync_*(result) C->>C: apply received state C->>UI: SignalBus.*_updated (signal down)
When the host player initiates, the SignalBus hop still fires locally but the RPC step is skipped — _apply_*() runs directly on the server.
