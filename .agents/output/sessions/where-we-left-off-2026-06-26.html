Pick Up Here #1 — UI: can't play a second card

Symptom: First card plays fine. After that, the UI won't let another card be played. No error in the console — so this is a UI / interaction-state problem, not a script crash.

Likely suspects to investigate (unconfirmed — start here):

  • A "card play enabled/disabled" flag (the player_card_play_enabled / listen_for_card_play_enabled/disabled RPC path on GameManager) that gets turned off on first play and never turned back on.
  • Input/selection state on the played card's UI node not being reset or re-armed after a successful play.
  • The phase that should re-enable play (e.g. back to MainPhase) not being re-entered, so the enable signal never re-fires.

To confirm: log the card-play-enabled flag on every flip and watch whether it ever returns to true after the first card resolves.

Pick Up Here #2 — Consume AP on card play

Goal: when a card is played, actually deduct its AP cost from the active player's action points. This is the input the turn-switch logic needs.

Why it matters: CheckTurnSwitchPhase decides to switch turns based on action_points.value < 0 (and SwitchTurnPhase advances active_player). Until AP is actually spent on play, that branch can never trip, so the turn never switches and we can't test the new check_turn_switch / switch_turn phases end to end.

Where to wire it: the successful-play path in game_manager.gd (around validate_card_play / card_played_succeeded) — subtract card.ap_cost from action_points when a play is validated.

Heads-up / cleanup nearby: validate_card_play still has the action_points.value + 10 >= card.ap_cost fudge — that +10 will mask real AP exhaustion and should come out once real consumption lands, or the switch threshold won't behave.

Test Path Once Both Are Done

  1. Play cards until AP goes negative.
  2. CheckTurnSwitchPhase should route to SwitchTurnPhase.
  3. SwitchTurnPhase rotates active_player via turn_order and hands off to on_complete_phase.
  4. Confirm the next player can now play, and the prior player can't.
