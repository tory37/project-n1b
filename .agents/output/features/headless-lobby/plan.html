FEATURE 2026-05-29

Headless Server Lobby

Overview

Adapts lobby_ui.gd and lobby_ui.tscn for a fully dedicated authoritative server model. The server always runs headless. When the lobby loads it auto-hosts immediately; once two clients connect it transitions to the game automatically. The Host and Start buttons are removed — clients never host, and the server never waits for a click.

Affected Files

src/ui/lobby/lobby_ui.gd src/ui/lobby/lobby_ui.tscn

NetworkManager and its API are untouched.

Key Decisions

No helper functions. DisplayServer.get_name() == "headless" is inlined directly in _ready() — one condition, no wrapper. After that, NetworkManager.is_server is the only check used anywhere else in the file.
Unit test scope: The new logic requires an active multiplayer peer and/or a live scene tree — both belong in integration tests. Testing is manual via the Doer Test Plan.
1 Remove Host & Start Buttons
  1. In lobby_ui.tscn, delete the HostButton node from Buttons and the StartButton node from VBoxContainer.
  2. In lobby_ui.gd, remove @onready var _host_button and @onready var _start_button.
  3. Remove their pressed.connect(…) calls and _start_button.visible = false from _ready().
  4. Delete _on_host(), _on_start(), and _update_start_button() entirely.
  5. Remove the _update_start_button() call from _on_player_list_changed().
  6. In _on_connection_failed(), remove the _host_button.disabled = false line (keep the join-button line).
Verification: Scene opens in the Godot editor without errors. No _host_button or _start_button references remain in the script.
2 Auto-Host & Auto-Start
  1. At the end of _ready(), add inline:
    if DisplayServer.get_name() == "headless": NetworkManager.host_game()
    Set _status_label.text to "Server running — waiting for players..." on success, error string on failure.
  2. Add func _check_auto_start() -> void: calls _start_game.rpc() when NetworkManager.connected_players.size() >= NetworkManager.MAX_PLAYERS.
  3. In _on_player_list_changed(), add: if NetworkManager.is_server: _check_auto_start().
  4. Update _on_connected() status text to "Connected! Waiting for game to start...".
Verification (manual):
  • Launching with --headless prints "Hosting game on port 7777" with no errors.
  • Two clients joining triggers an automatic scene transition — no button press needed.
  • One client joining does not trigger a scene transition.
  • A non-headless launch shows only the Join UI with no auto-host side effects.
