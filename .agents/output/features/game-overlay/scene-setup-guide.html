Scene Setup Guide: game_overlay_ui.tscn

Goal

A simple HUD overlay that floats over the 3D game scene and displays live game state: turn, active player, AP, currency, and hand/deck/discard counts for each player. It reads from SignalBus signals and updates Labels in place — no fancy styling, just readable output to prove the data pipeline is working.

You perform all of the editor work below. The script changes are separate and will be presented for your approval after these steps are complete.

Godot UI Concepts You Need

Control nodes vs Node3D

Every visible 2D UI element in Godot inherits from Control. Control nodes use an anchor + offset system to position themselves relative to their parent — not the 3D world. Anchors are fractions of the parent's size (0.0 = left/top, 1.0 = right/bottom). anchors_preset = 15 in the lobby scene means "Full Rect" — stretch to fill the entire parent.

CanvasLayer — why you need it for a HUD

When your scene root is Node3D (like game.tscn), adding a Control child directly works, but gives you no explicit control over draw order. A CanvasLayer node creates its own isolated rendering layer. Set its Layer property to any number above 0 and it is guaranteed to render on top of the 3D world. This is the standard pattern for HUDs in Godot 4.

Layout containers

Godot containers automatically size and position their children — you don't set positions manually.

ContainerWhat it does
PanelContainerDraws a themed background panel; sizes to its content
MarginContainerAdds inner padding around its single child
VBoxContainerStacks children vertically with a configurable gap

You nest them: Panel → Margin → VBox → Labels.

Scene Unique Names (%NodeName)

Instead of referencing a node by its full path ($Panel/Margin/Layout/TurnLabel), you can flag any node as a Scene Unique Name and then access it from the script as %TurnLabel. This survives renames anywhere in the path and is the clean, refactor-safe way to reference UI nodes from a script.

Step-by-Step Editor Instructions

1 Change the root node type to CanvasLayer
  1. Open src/ui/game_overlay/game_overlay_ui.tscn in the Godot editor.
  2. The current root node is a bare Control. Right-click GameOverlayUi in the Scene panel → Change Type.
  3. Search for CanvasLayer → click Change.
  4. In the Inspector, find the Layer property and set it to 2.
2 Add PanelContainer
  1. Select GameOverlayUi (the CanvasLayer root).
  2. Click the + (Add Child Node) button → search PanelContainer → Create.
  3. Rename it to Panel.
  4. In the Inspector, under Control → Layout → Custom Minimum Size, set x = 240.
  5. Leave the Anchor preset at default (top-left). In the Inspector under Control → Layout → Offset, set Left = 12 and Top = 12 to give it a small margin from the screen corner.
3 Add MarginContainer
  1. Select Panel.
  2. Add child → MarginContainer → rename to Margin.
  3. In the Inspector under Theme Overrides → Constants, set all four margin values to 8:
    • margin_left = 8
    • margin_right = 8
    • margin_top = 8
    • margin_bottom = 8
4 Add VBoxContainer
  1. Select Margin.
  2. Add child → VBoxContainer → rename to Layout.
  3. In the Inspector under Theme Overrides → Constants → Separation, set to 4.
5 Add Labels and Separators

Select Layout and add each child in order. For each Label, set its placeholder text in the Inspector's Text field — the script will overwrite these at runtime, but the placeholder helps you see the layout while editing.

6 Attach the script
  1. Select the root GameOverlayUi node.
  2. In the Inspector, the Script property may have been cleared when you changed the node type. If so, drag res://src/ui/game_overlay/game_overlay_ui.gd onto the Script slot in the Inspector.
7 Enable Scene Unique Names on every Label

The script will use %TurnLabel syntax to find nodes without fragile path strings. You must opt each Label into this system:

  1. Right-click each of the 12 Labels in the Scene panel.
  2. Click Access as Scene Unique Name.
  3. A % icon will appear next to the node — that confirms it worked.

Do this for all 12 Labels. The two HSeparators do not need it.

8 Instantiate into game.tscn
  1. Open src/levels/game/game.tscn.
  2. Select the root Game node.
  3. In the top menu: Scene → Instantiate Child Scene… → select game_overlay_ui.tscn → Open.
  4. The overlay node will appear in the scene tree. No repositioning needed — CanvasLayer children are always fixed to the screen.
Checkpoint: Run the game. You should see a small dark panel in the top-left corner with placeholder text ("Turn: 0", "AP: 0", etc.). Nothing updates yet — that happens after the script is wired up.

Why the lobby didn't use CanvasLayer

The lobby scene (lobby_ui.tscn) has a Control root with anchors_preset = 15 (Full Rect). That's correct for the lobby because the lobby IS the entire scene — there's no 3D world behind it. The lobby's root viewport is the canvas, so Control fills it directly.

The game overlay is different: it floats on top of a Node3D scene. That's exactly what CanvasLayer is designed for.
