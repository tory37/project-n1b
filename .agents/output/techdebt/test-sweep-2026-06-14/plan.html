TECHDEBT 2026-06-14

Test Suite Clean Sweep

Audit Summary

The test suite was written against a now-defunct GameManager monolith and the old PlayerState / PlayerSeat types. The codebase has since migrated to a component architecture with networked RPC state and no PlayerSeat enum. Six test files exist; three need to be rewritten entirely, two need targeted fixes, and one (test_hex_utils.gd) is still valid as-is.

File-by-File Findings

FIX test_finite_state_machine.gd

FiniteState.enter() now has the signature enter(_payload: Variant). The inner TrackedState class overrides it with no parameters, which causes a runtime call error when FiniteStateMachine.change_state() invokes enter(payload).

Fix: Update TrackedState.enter() to func enter(_payload: Variant) -> void:. No other changes needed — the FSM logic itself is unchanged.

REWRITE test_signal_bus.gd

All four tests reference signals that no longer exist (player_switched, ap_moved). The new SignalBus has 16 signals; the tests cover 0 of them.

New coverage: Assert existence and correct emission of every signal in the current SignalBus. No setup is required — SignalBus is an autoload.

FIX test_local_session.gd

All tests reference PlayerSeat.PLAYER_ONE / PLAYER_TWO, an enum that no longer exists. LocalSession.local_player_id is now a plain int defaulting to 0.

Fix: Replace every PlayerSeat.PLAYER_ONE with 1 and PlayerSeat.PLAYER_TWO with 2. Update the default-value test to expect 0 (the actual default), and set local_player_id = 1 in before_each.

REPLACE test_player_game_state.gd

Tests a PlayerState class (deck_to_hand(), use_card_from_hand()) that no longer exists. Player state is now split into GameCardCollectionComponent nodes inside NetworkedPlayer.

Replacement: Rename to test_game_card_collection.gd and test GameCardCollection (a pure Resource — no scene tree required). This class contains all the card-moving logic that was previously in PlayerState.

REPLACE test_game_manager.gd

Every test references properties and methods that no longer exist on GameManager (ap, active_player, player_state, max_ap_value, base_income, turn_phase_fsm, _on_*_requested() handlers, PlayerSeat enum). The new GameManager delegates all state to component nodes and requires a full Godot scene tree — it cannot be unit-tested in isolation.

Replacement strategy: Delete the monolithic file and create four focused test files — one per state component that can be tested:

  • test_action_points_component.gd
  • test_active_player_component.gd
  • test_turn_order_component.gd
  • test_round_number_component.gd

Each component extends Node and uses multiplayer.is_server() guards. In a GUT context with no multiplayer peer, Godot treats the local peer as server (peer ID = 1), so all server-gated paths execute. RPC functions decorated with "call_local" execute locally without a network peer. Tests will call methods directly and assert on .value.

OK test_hex_utils.gd

All tests are valid. HexUtils API is unchanged. No action needed.

Notes on Multiplayer in Unit Tests

Components extend Node and use multiplayer.is_server() guards and @rpc(..., "call_local", ...) sync functions. In GUT's offline context:

  • is_server() → true — Godot treats local-only as server (peer ID 1). All server-gated paths execute.
  • .rpc() with "call_local" — executes locally. No network peer required.
  • Signal emission — _sync_value functions gate on if not is_server() before emitting to SignalBus. In tests, signals won't fire via the RPC path. Test values, not signal emissions, for these components.
  • add_child_autofree — use this to give Node components a valid tree context.

If RPC calls generate noise warnings during test runs, that's expected. They don't indicate test failures.
